Tools Agent & n8n Fundamentals

n8n hat seine AI-Agent-Architektur drastisch vereinfacht. Der Tools Agent ist nun der Standard für alle Implementierungen. Lerne die fundamentale Architektur und warum diese Vereinfachung entscheidend ist.

Die Tools Agent Revolution

Seit Version 1.82.0 hat n8n eine fundamentale Vereinfachung vorgenommen. Ursprünglich existierten mehrere Agent-Typen (Conversational Agent, ReAct Agent, OpenAI Functions Agent), doch ab Februar 2025 wurde die Architektur konsolidiert.

✨ Warum Tools Agent?
  • Verbesserte Tool-Calling-Fähigkeiten: Zuverlässigere Ausführung
  • Standardisierte Output-Formate: Konsistente Ergebnisse
  • LangChain-basiert: Production-ready Tool-Interface
  • Multi-Model-Support: OpenAI, Groq, Mistral, Anthropic, Azure

Was wurde entfernt?

  • ReAct Agent: Limitation war fehlendes Memory-Support
  • Conversational Agent: Funktionalität ist im Tools Agent integriert
  • OpenAI Functions Agent: Nur noch für spezielle Use Cases
⚠️ Migration

Workflows die auf "Tools Agent" konfiguriert waren, funktionieren mit der aktualisierten Node-Version weiterhin. Ältere Agent-Typen müssen manuell migriert werden.

Die zwei Hauptkomponenten

Die Prompt-Struktur in n8n folgt einer klaren Hierarchie mit zwei Hauptkomponenten:

  1. System Message: Definiert Verhalten und steuert alle Agent-Entscheidungen
  2. User Prompt: Enthält die tatsächliche Anfrage des Users

Diese Trennung ist fundamental für effektive Agents. Die System Message "primed" das Model BEVOR es den User Input verarbeitet.

💡 Key Takeaway

System Message = WIE der Agent antwortet
User Message = WAS beantwortet wird
